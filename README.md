# 智能座舱街景理解系统
### 📺 [点击此处下载观看：智能座舱街景理解系统演示视频](https://github.com/zhangyuxuan0306-gif/Vehicle-Scene-Interaction-System/releases/download/V1.0/22.mp4)
本系统是一款基于**多模态大模型（VLM/LLM）与前沿深度学习技术**构建的下一代**智能座舱车载场景交互与感知系统**。项目打通了舱内“人眼/手势”行为感知与舱外“实时街景”目标检测的物理边界，实现了真正意义上的“人-车-路-景”多模态智能协同交互。

## 🚀 核心功能与运行机制

### 1. 舱外街景感知与多模态交互
* **车载街景目标检测**：系统基于自训练的 **YOLO 目标检测模型**，针对国贸等城市典型 CBD 区域的 7 类主要建筑（如特定地标、写字楼等）进行高精度的实时多目标跟踪与识别。
* **座舱交互意图捕捉（红点落点机制）**：系统在 Web UI 与视频流交互中实现了**“所见即所点”**的交互原型。乘客可通过在界面上直接**点击选中**视线所及的建筑目标，系统会精准捕捉该点击位置并生成交互“红点”（像素焦点坐标）。
  * *设计前瞻性*：当前的“鼠标点击红点”在未来可直接无缝等效替代为车载硬件的**视线追踪（Gaze）注视区域**或**手势指引（Gesture）的激光投影点**。
* **空间实体匹配**：系统后台通过将交互红点的几何坐标与 YOLO 实时生成的建筑目标检测框（Bounding Box）进行动态空间求交匹配，从而精准、毫秒级地锁定乘客当前想要交互的特定建筑实体。

### 2. 两阶段混合检索问答引擎 (Local Q&A Engine)
当用户针对窗外街景（如某栋建筑）进行提问时，系统具备极强的健壮性与长尾问题处理能力，采用以下**两阶段智能兜底策略**进行感知响应：
* **第一阶段：本地精准匹配**
  系统优先检索车载本地的结构化知识库（如 `buildings.json`），若成功命中，则提取相关的地理位置、建筑背景、周边评测等数据，由大模型快速组织语言进行高时效的局部回复。
* **第二阶段：联网动态检索与 RAG 提炼（本地缺失兜底）**
  若用户的提问涉及冷门场景、实时路况、突发新闻或超出了本地库的知识边界，系统将自动触发**联网搜索机制**。系统利用搜索引擎 API 针对当前识别到的实体动态抓取多源网页的实时动态信息，并将其作为上下文输入给大语言模型。大模型利用强大的文本压缩与信息抽取能力，自动去除广告和网页噪声，对碎片化文本进行深度清洗、逻辑整理与重组润色，最终为用户平滑输出结构清晰、切中要害的高质量精简回答。

## 快速开始

```bash
source venv/bin/activate
python main.py --mode video          # 视频点击识别（推荐）
python main.py --mode ui             # Web 界面
```
